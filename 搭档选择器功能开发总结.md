# 搭档选择器功能开发总结

## 一、功能概述

在 Chatbox 会话页面（`session/$sessionId.tsx`）添加了一个搭档选择器下拉菜单，允许用户在不离开当前会话的情况下快速切换 AI 搭档。

### 核心功能
1. **搭档选择器 UI** - 在会话页面 Header 下方显示一个醒目的渐变按钮
2. **下拉菜单** - 显示"我的搭档"（本地存储）和"远程搭档"（服务器获取）列表
3. **切换搭档** - 选择搭档后自动：
   - 更新会话的 copilotId、picUrl、name
   - 更新 system message（新搭档的提示词）
   - 自动发送消息触发大模型回复

## 四、交互逻辑详解

### 4.1 切换搭档完整流程

```
用户点击搭档按钮
       ↓
展开下拉菜单（显示我的搭档 + 远程搭档）
       ↓
用户选择新搭档（如 "04 检索表达式构建"）
       ↓
handleSelectCopilot(copilot) 执行：
  1. 检查是否有消息正在生成，如有则取消
  2. 查找当前会话的 system message
  3. 用新搭档的 prompt 替换 system message 内容
  4. 调用 updateSessionStore 更新会话信息
  5. 延迟 500ms
  6. 自动发送用户消息："你好，04 检索表达式构建！"
  7. 大模型收到消息 + 新的 system prompt
  8. 大模型基于新搭档的角色生成回复
```

### 4.2 关键代码逻辑

```typescript
const handleSelectCopilot = useCallback(async (copilot: CopilotDetail) => {
  // 1. 停止当前生成
  if (lastGeneratingMessage?.generating) {
    lastGeneratingMessage?.cancel?.()
  }
  
  // 2. 获取当前消息列表
  const currentMessages = currentSession.messages || []
  
  // 3. 找到或创建 system message
  const systemMsgIndex = currentMessages.findIndex(m => m.role === 'system')
  const newSystemMessage = {
    id: systemMsgIndex >= 0 ? currentMessages[systemMsgIndex].id : crypto.randomUUID(),
    role: 'system' as const,
    contentParts: [{ type: 'text' as const, text: copilot.prompt }],
  }
  
  // 4. 更新 messages 数组
  if (systemMsgIndex >= 0) {
    newMessages[systemMsgIndex] = newSystemMessage
  } else {
    newMessages.unshift(newSystemMessage)
  }
  
  // 5. 更新 session
  updateSessionStore(currentSession.id, {
    name: copilot.name,
    picUrl: copilot.picUrl,
    copilotId: copilot.id,
    messages: newMessages,
  })
  
  // 6. 延迟发送用户消息触发回复
  setTimeout(async () => {
    const userMessage: Message = {
      id: crypto.randomUUID(),
      role: 'user',
      contentParts: [{ type: 'text', text: '你好，' + copilot.name + '！' }],
      timestamp: Date.now(),
    }
    await submitNewUserMessage(currentSession.id, {
      newUserMsg: userMessage,
      needGenerating: true,
    })
  }, 500)
}, [currentSession, lastGeneratingMessage])
```

## 五、调试过程记录

### 5.1 遇到的问题及解决方案

#### 问题 1: 搭档选择器不显示
**现象**: 代码添加后页面上看不到选择器  
**调试方法**: 
- 添加 `console.log` 调试日志
- 使用简单的 `<div>` 测试元素排查渲染问题

**原因**: 可能是条件渲染问题或 CSS 隐藏

**解决**: 
1. 先添加简单的测试 div（红色边框黄色背景）
2. 确认渲染正常后逐步替换为正式 UI

#### 问题 2: TypeScript 错误 - "messages" 属性不存在
**错误信息**:
```
对象字面量只能指定已知属性，并且"messages"不在类型"Updater<Omit<Session, 'id'>>"中
```

**原因**: `updateSessionStore` 的类型定义不允许直接更新 `messages`

**状态**: 运行时仍然正常工作（类型定义与实际实现不一致）

#### 问题 3: 切换搭档后页面不停跳动
**现象**: 选择搭档后会话标题不停变化，页面闪烁

**原因**: 
- 切换搭档时更新了 `name` 属性
- Header 组件中有自动生成标题的逻辑 (`scheduleGenerateNameAndThreadName`)
- 每次 name 更新都会触发自动标题生成

**解决**: 
- 不更新 `name` 属性，只更新 `copilotId` 和 `picUrl`
- 但这导致搭档名称显示不正确

**最终方案**: 
- 保留更新 `name`，但更新后的流程优化（见问题4）

#### 问题 4: 切换搭档后没有触发大模型回复
**现象**: 选择搭档后只是更新了名称，但大模型没有响应

**原因**: 只更新了会话信息，没有发送消息给大模型

**解决**: 
- 自动发送一条"你好，xxx！"的用户消息
- 延迟 500ms 执行，确保 session 更新完成
- 大模型收到消息后基于新 system prompt 回复

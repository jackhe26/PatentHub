# Header 拖拽与搭档选择器位置优化开发总结

**日期**：2026-06-11
**作者**：PatentHub 开发团队
**类型**：UI 交互修复

---

## 一、问题背景

用户反馈 PatentHub 桌面应用顶部 Header 区域存在三个问题：

1. **窗口不能通过鼠标拖拽移动**：在 Header 顶部空白处按住鼠标拖动，软件窗口没有反应
2. **搭档选择器与 Header 不在同一行**：UI 不符合用户原本的设计意图
3. **Header 右侧空白区域不能拖拽**：只有左侧空白可以拖拽，右侧不行

---

## 二、问题分析与修复

### 问题 1：Toolbar 容器覆盖拖拽区域

**根本原因**

`src/renderer/components/layout/Toolbar.tsx` 第 76 行的 Flex 容器上错误地添加了 `className="controls"`：

```tsx
// 错误代码
<Flex align="center" gap="md" className="controls">
  {showUpdateNotification && <UpdateAvailableButton />}
  ...
</Flex>
```

CSS 规则（`src/renderer/static/index.css`）会自动将所有 `.title-bar .controls` 标记为 `no-drag`：

```css
.title-bar .controls,
.title-bar button,
.title-bar [role="button"],
... {
  -webkit-app-region: no-drag !important;
}
```

由于 `class="controls"` 作用在整个 Flex 容器上，**容器覆盖的整个区域**（包括按钮之间的空白）都被标记为 `no-drag`，导致拖拽失效。

**修复**

移除 `Toolbar.tsx` Flex 容器上的 `className="controls"`：

```tsx
// 修复后
<Flex align="center" gap="md">
  {showUpdateNotification && <UpdateAvailableButton />}
  ...
</Flex>
```

内部的可点击元素（Button、ActionIcon）仍会被通用规则（`button`/`[role="button"]`）自动识别为 `no-drag`。

---

### 问题 2：搭档选择器与 Header 不在同一行

**原问题结构**

`$sessionId.tsx` 中有一个独立的 `<Box>` 容器作为"搭档选择器行"：

```tsx
// 旧代码
<Box className="title-bar ..." style={{ WebkitAppRegion: 'drag' }}>
  <ActionIcon>{PDF 切换按钮}</ActionIcon>
  <div style={{ marginLeft: '35%' }}>  // ← 强行居中
    <Menu>我的审查搭档</Menu>
  </div>
</Box>
<Header session={currentSession} />  ← Header 被挤到下面
```

**问题**：
- 多占一行高度
- `marginLeft: '35%'` 把按钮推到中央，导致这一行几乎没有可拖拽空白
- 与 Header 分离，UI 不够紧凑

**修复方案**

1. **重构 Header 组件**，添加 `leftActions` 和 `copilotSelector` 两个可选 props：

```tsx
type HeaderProps = {
  session: Session
  leftActions?: ReactNode      // 渲染在侧边栏和 Session Name 之间（如 PDF 按钮）
  copilotSelector?: ReactNode  // 中央位置渲染的搭档选择器
}
```

2. **$sessionId.tsx 移除独立 Box**，将 PDF 按钮和搭档 Menu 作为 props 传入 Header：

```tsx
<Header
  session={currentSession}
  leftActions={<ActionIcon>{PDF 切换按钮}</ActionIcon>}
  copilotSelector={
    <Menu>
      <Menu.Target>
        <Button>我的审查搭档</Button>
      </Menu.Target>
      ...
    </Menu>
  }
/>
```

**效果**：
- 顶部只有一行 Header，UI 更紧凑
- 拖拽逻辑统一在 Header 管理
- 搭档选择器自然居于 Session Name 之后

### 问题 3：Header 右侧空白不能拖拽

**根本原因**

修复问题 2 后，Header 右侧的容器用 `flex: '1 1 0'` 撑满空白：

```tsx
// 旧代码
<Flex style={{ flex: '1 1 0', minWidth: 0, justifyContent: 'flex-end' }}>
  <Toolbar />
  <WindowControls />
</Flex>
```

这个 Flex 容器**本身不是按钮**，但：
- `flex: '1 1 0'` 让它撑满搭档按钮到 WindowControls 之间的所有空间
- 容器有 `pointer-events: auto`，会拦截鼠标事件
- 结果：搭档按钮右侧的所有空白都被这个容器"占据"，外层 `title-bar` 的 `drag` 区域被覆盖

**修复**

将 `flex: '1 1 0'` 改为 `marginLeft: 'auto'` + `flexShrink: 0`：

```tsx
// 修复后
<Flex style={{ marginLeft: 'auto', flexShrink: 0 }}>
  <Toolbar />
  <WindowControls />
</Flex>
```

**核心区别**：
- `flex: 1`：容器扩展填满剩余空间，**整个容器区域都是 DOM 元素**
- `marginLeft: auto`：只添加左侧 margin 推到右边，**容器只包裹子元素大小**

这样容器与 WindowControls 之间的空白回归到外层 `title-bar` 的 `drag` 范围，可以正常拖拽。

---

### 问题 4：搭档按钮位置优化

**用户反馈**：搭档按钮应该放在 PDF 区域和对话窗口的中间，而不是最右侧。

**修复**：限制 Session Name 的 `maxWidth`，让搭档按钮自然居中靠左：

```tsx
<Flex style={{
  flex: '1 1 0',
  minWidth: 0,
  maxWidth: isSmallScreen ? '50%' : '35%',  // ← 关键
}}>
  <Title>{currentSession?.name}</Title>
  <ActionIcon>编辑按钮</ActionIcon>
</Flex>

{copilotSelector && <Flex style={{ flexShrink: 0 }}>{copilotSelector}</Flex>}
```

**效果**：
- Session Name 最多占 35% 宽度（长名称自动截断）
- 搭档按钮自然跟在 Session Name 之后
- 在 PDF 区域（45%）和对话区域（55%）的分界附近
- 不浪费中间空白
- 小屏自动放宽到 50%

## 四、关键技术经验

### 1. Electron frameless 窗口拖拽规则

- `-webkit-app-region: drag` 让区域可拖拽
- `.controls` 类（以及 `button`/`[role="button"]`）自动标记为 `no-drag`
- **坑1**：给整个容器加 `controls` 类会让容器内所有空白也变成 `no-drag`
- **坑2**：`flex: 1` 的容器撑满空白时，容器的 DOM 区域会拦截 `drag` 事件

### 2. `flex: 1` vs `marginLeft: auto`

| 特性 | `flex: 1` | `marginLeft: auto` |
|---|---|---|
| 容器宽度 | 撑满剩余空间 | 只包裹子元素 |
| 空白归属 | 容器 DOM 区域 | 父元素背景 |
| 事件穿透 | 容器拦截 | 父元素接收 |
| 视觉效果 | 元素靠右 | 元素靠右 |

**结论**：想让元素靠右但保留父级拖拽区域，用 `marginLeft: auto`。

### 3. Header 重构为 props 注入

- 把特定页面的 UI（搭档选择器、PDF 按钮）通过 `ReactNode` props 注入通用 Header
- 优点：Header 保持通用、拖拽逻辑集中管理，页面自定义无需修改 Header 内部
- 注意：`Session Name` 区域用 `maxWidth` 限制宽度，防止长名称把右侧元素挤到边缘

### 4. 诊断思路

1. **截图定位**：先确认哪些区域可以拖拽、哪些不行
2. **搜索关键 CSS**：`WebkitAppRegion` / `controls` / `title-bar` 找到所有相关代码
3. **逐层缩小范围**：从外层 `title-bar` → 中间 Flex 容器 → 内部按钮元素
4. **容器事件拦截**：`flex: 1` 容器虽然视觉上"空"，但实际是 DOM 元素，拦截了所有事件

# 图片比例与分辨率功能开发总结

## 背景

图片生成模块原先只支持 4 种图片比例（auto、1:1、3:2、2:3），且 `aspectRatio` 参数在 `AbstractAISDKModel.paint()` 中未实际传递给 AI SDK 的 `generateImage()` API，导致比例选择无效。

## 改动概述

| 文件 | 改动 |
|------|------|
| `constants.ts` | 比例从 4 个扩展到 14 个；新增 `RESOLUTION_OPTIONS`、`SIZE_MAP`、`getImageSize()` |
| `index.tsx` | 工具栏增加分辨率选择器 UI；`handleSubmit` 计算 `size` 并传递 |
| `imageGenerationActions.ts` | `GenerateImageParams` 增加 `size` 字段 |
| `abstract-ai-sdk.ts` | `paint()` 接收 `size` 并传给 `generateImage()` |
| `types.ts` | `ModelInterface.paint` 接口添加 `size` 参数 |

## 关键设计

### 数据流

```
UI 选择: 16:9 + 2k
  → getImageSize("16:9", "2k") = "2560x1440"
    → createAndGenerate({ aspectRatio: "16:9", size: "2560x1440" })
      → model.paint({ aspectRatio: "16:9", size: "2560x1440" })
        → generateImage({ size: "2560x1440" })
```

### 支持的完整比例列表

`auto`, `1:1`, `3:2`, `2:3`, `4:3`, `3:4`, `5:4`, `4:5`, `16:9`, `9:16`, `2:1`, `1:2`, `21:9`, `9:21`

### 分辨率选项

`1k`（默认）、`2k`、`4k`

### 兼容性说明

- 比例为 `auto` 时不传 `size`（由 API 自行决定尺寸）
- 所有模型通用比例列表（不再按模型区分）
- 模型不支持所选分辨率时返回 API 错误，由现有错误处理机制处理

# 图片生成快捷指令翻译开发总结

## 概述

本次开发任务是为 PatentHub 的图片生成模块（Image Creator）中的快捷指令（Quick Prompts）添加完整的国际化支持。快捷指令是图片生成界面 EmptyState 组件中显示的默认提示词卡片，用户点击后可直接填充提示词输入框，快速生成特定风格的图片。

## 双层提示词架构（核心设计亮点）

这是本模块最关键的架构设计，分为两层：

### 第 1 层：界面显示 — 简短的本地化文本
- **定义**: `displayKey` 字段，指向 i18n 翻译键
- **渲染**: `t(item.displayKey)` 通过 `useTranslation()` 渲染
- **作用**: 用户界面上看到的短标题（按钮上的简洁文案）
- **示例**: `"黄仁勋直播"` (zh) / `"Jensen Huang direct"` (en) / `"Jensen Huang in diretta"` (it)

### 第 2 层：实际发送 — 完整的英文扩充提示词 + 语言指令
- **定义**: `expanded` 字段，固定英文的完整提示词
- **渲染**: `fullPrompt = ${item.expanded}\n\n${getLanguageInstruction()}`
- **作用**: 点击按钮时，实际发送给 AI 的完整 Prompt，包含：
  1. **英文扩充提示词**（`expanded`）：详细描述图片场景、构图、风格、色彩、比例等，英文对 AI 理解效果最好
  2. **语言指令**（`getLanguageInstruction()`）：根据当前系统语言自动追加，让 AI 生成的图片文字内容与用户系统语言一致

```
示例 zh-Hans 用户点击 "黄仁勋直播" 时，实际发送给 AI 的是：
  [英文详细扩充提示词]
  
  Important: 图片中的所有文字内容请使用简体中文。
```

## 涉及的模块

### 1. EmptyState 组件
- **路径**: `src/renderer/routes/image-creator/-components/EmptyState.tsx`
- **功能**: 展示图片创建器的空状态界面，包含引导文案和快捷指令卡片
- **关键机制**:
  ```typescript
  const QUICK_PROMPTS = [
    {
      displayKey: 'quick_prompt_huang',  // ① 界面显示简短的本地化文本
      expanded: `Generate a realistic...`, // ② 实际发送完整的英文扩充提示词
    },
    // ...
  ]
  
  // 点击时构造完整prompt
  const fullPrompt = `${item.expanded}\n\n${getLanguageInstruction()}`
  ```

## 开发过程中发现的问题

### 问题 1：硬编码的中文提示词

在 `src/renderer/routes/image-creator/index.tsx` 中，存在 5 个**硬编码中文提示词**字符串，用于 "Create a New Image-Creator Conversation" 按钮下方的示例提示：

```typescript
const images = [
  "赛博朋克风格的城市夜景，霓虹灯闪烁，雨夜的街道反射着彩色灯光",
  // ...
]
```

这些字符串被 `translateImages()` 函数使用，在生成新图片会话时作为默认提示词随机选择。**这是未国际化的问题** — 非中文用户看到的始终是中文提示词。

### 问题 2：EmptyState 与 index.tsx 的提示词是不同用途

- `EmptyState.tsx` 中的快捷指令 → 用于**空状态界面指引**，是固定的 5 个主题卡片，有**双层机制**（显示简短/发送完整）
- `index.tsx` 中的硬编码图片 → 用于**创建新会话时的随机默认提示词**，是另一套内容，且没有双层机制

两者不应混淆，需要独立处理。但 index.tsx 也需要国际化。

### 问题 3：翻译文件格式一致性

各语言文件的 JSON 格式需要保持完全一致（键的顺序、是否有尾逗号等），否则在 i18n 解析时可能出错。

## 经验总结

### 1. 图片生成提示词设计模式（「显示简短+发送完整」）

```
┌─────────────────────────────────────────────────┐
│  用户界面看到的是： "黄仁勋直播"（简短本地化文本）   │
│                    ↓ 点击卡片                       │
│  实际发送给AI的是：                                  │
│  "Generate a realistic Douyin live streaming...    │
│   [数百字详细英文描述]                                │
│                                                   │
│  最后追加语言指令：                                  │
│   Important: 图片中的所有文字内容请使用简体中文。"     │
└─────────────────────────────────────────────────┘
```

这种设计的好处：
- **界面简洁**: 用户看到的是简洁易懂的短标签
- **AI 理解准确**: 发送给 AI 的 `expanded` 用英文编写，AI 对英文提示词理解最精准
- **语言自适应**: 通过 `getLanguageInstruction()` 追加语言指令，让生成的图片文字内容适配用户系统语言
- **可维护性强**: 翻译只需维护短标签（`displayKey`），`expanded` 提示词统一用英文

### 2. 国际化最佳实践

- 界面显示层 → 通过 i18n 翻译（`t(item.displayKey)`）
- AI 输入层 → 用英文编写固定提示词（`expanded`）
- 语言适配层 → 用语言指令映射表追加（`getLanguageInstruction()`）

### 3. 多语言维护策略

该应用支持 **15+ 语言**，涵盖：简体中文、繁体中文、日语、韩语、英语、德语、法语、西班牙语、意大利语、葡萄牙语、瑞典语、挪威语、俄语、阿拉伯语。

### 4. 后续待办项

> **⚠️ 需要继续完成**：将 `index.tsx` 中的 5 个硬编码中文提示词也替换为 i18n 翻译，添加对应的翻译键到所有 16 个语言文件中，让创建新会话时的随机默认提示词也支持国际化。

## 日期

2025-04-26（初版）

## 一、概述

### 1.1 功能背景

图片生成功能是 PatentHub 应用中允许用户通过 AI 模型生成图片的能力。用户可以选择不同的 AI 模型来生成图片，涉及到模型选择、API 调用、图片数据处理等多个层面。

### 1.2 遇到的问题

1. **模型列表问题**：某些内置模型（如 Chatbox AI 的 GPT Image、Google Gemini 的 Nano Banana）在应用中无法正常工作
2. **图片格式问题**：部分第三方 API 返回图片时使用 URL 格式，而非 Base64 格式，导致前端 `atob()` 解码失败
3. **模型能力识别**：无法准确识别哪些模型支持图片生成能力
4. **自定义 Provider 问题**：自定义 OpenAI 兼容 API 的模型无法正常生成图片（404 错误 + atob 解码失败）

## 三、核心代码文件

### 3.1 abstract-ai-sdk.ts（核心逻辑层）

**文件路径**: `src/shared/models/abstract-ai-sdk.ts`

**职责**: 
- 实现图片生成的抽象方法 `paint()`
- 处理 AI SDK 返回的图片数据
- 兼容 Base64 和 URL 两种返回格式

**核心方法**:

```typescript
public async paint(
  params: {
    prompt: string
    images?: { imageUrl: string }[]
    num: number
    aspectRatio?: string
  },
  signal?: AbortSignal,
  callback?: (picBase64: string) => void
): Promise<string[]> {
  const imageModel = this.getImageModel()
  if (!imageModel) {
    throw new ApiError('Provider doesnt support image generation')
  }
  
  const result = await generateImage({
    model: imageModel,
    prompt: params.prompt,
    n: params.num,
    abortSignal: signal,
  })

  // 处理图片数据，支持 base64 或 URL 两种格式
  const dataUrls = await Promise.all(
    result.images.map(async (image) => {
      // 优先使用 base64
      if (image.base64) {
        // 检查 base64 字符串是否实际上是 URL（某些 API 返回 URL 作为 base64 字段）
        const base64Str = image.base64.trim()
        if (base64Str.startsWith('http://') || base64Str.startsWith('https://')) {
          // 这是一个 URL，不是 base64，需要下载
          try {
            const response = await fetch(base64Str)
            const blob = await response.blob()
            const arrayBuffer = await blob.arrayBuffer()
            const base64 = btoa(
              new Uint8Array(arrayBuffer).reduce((data, byte) => data + String.fromCharCode(byte), '')
            )
            return `data:${image.mediaType};base64,${base64}`
          } catch (error) {
            console.error('Failed to download image from URL:', base64Str, error)
            throw new ApiError(`Failed to download image from URL: ${base64Str}`)
          }
        }
        // 正常的 base64 数据
        return `data:${image.mediaType};base64,${image.base64}`
      } else {
        throw new ApiError('Image generation result does not contain base64 data')
      }
    })
  )

  for (const dataUrl of dataUrls) {
    callback?.(dataUrl)
  }
  return dataUrls
}
```

**关键点**:
- 使用 Vercel AI SDK 的 `experimental_generateImage` 函数
- `getImageModel()` 是抽象方法，由子类实现
- 处理两种返回格式：Base64 直接使用，URL 需要下载

### 3.3 custom-openai.ts（自定义 API 适配器）⚠️ 2025-04-27 新增

**文件路径**: `src/shared/providers/definitions/models/custom-openai.ts`

**职责**:
- 处理用户自定义的 OpenAI 兼容 API
- 直接调用图片生成 API，绕过 AI SDK
- 正确处理返回 URL 的模型

**核心方法 - paint()**:

```typescript
/**
 * 重写 paint 方法，直接调用 API 绕过 AI SDK
 * 这样可以正确处理返回 URL 的模型（如 nano-banana）
 */
public async paint(
  params: {
    prompt: string
    images?: { imageUrl: string }[]
    num: number
    aspectRatio?: string
  },
  signal?: AbortSignal,
  callback?: (picBase64: string) => void
): Promise<string[]> {
  const modelId = this.options.model.modelId

  // 构建请求体
  const requestBody: Record<string, unknown> = {
    model: modelId,
    prompt: params.prompt,
    n: params.num,
  }

  // 添加图片比例支持
  if (params.aspectRatio) {
    const [width, height] = params.aspectRatio.split(':').map(Number)
    if (width && height) {
      const baseSize = 1024
      if (width === height) {
        requestBody.size = '1024x1024'
      } else if (width > height) {
        requestBody.size = `${baseSize}x${Math.round(baseSize * (height / width))}`
      } else {
        requestBody.size = `${Math.round(baseSize * (width / height))}x${baseSize}`
      }
    }
  }

  // 直接用 fetch 调用图片生成 API
  const fetchFn = createFetchWithProxy(this.options.useProxy, this.dependencies)
  const url = this.buildImageGenerationUrl()  // 智能构建 URL

  const response = await fetchFn(url, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${this.options.apiKey}`,
    },
    body: JSON.stringify(requestBody),
    signal,
  })

  if (!response.ok) {
    const errorText = await response.text()
    throw new ApiError(`Image generation failed: ${response.status} ${response.statusText} - ${errorText}`)
  }

  const result = await response.json() as {
    data: Array<{
      b64_json?: string
      url?: string
      revised_prompt?: string
    }>
  }

  if (!result.data || result.data.length === 0) {
    throw new ApiError('No images generated')
  }

  // 处理每张图片
  const dataUrls: string[] = []
  for (const image of result.data) {
    let dataUrl: string

    if (image.b64_json) {
      // 直接返回 base64
      dataUrl = `data:image/png;base64,${image.b64_json}`
    } else if (image.url) {
      // 下载 URL 并转换为 base64
      try {
        const imageResponse = await fetch(image.url)
        if (!imageResponse.ok) {
          throw new Error(`Failed to download image: ${imageResponse.status}`)
        }
        const blob = await imageResponse.blob()
        const arrayBuffer = await blob.arrayBuffer()
        const base64 = btoa(
          new Uint8Array(arrayBuffer).reduce((str, byte) => str + String.fromCharCode(byte), '')
        )
        dataUrl = `data:${blob.type || 'image/png'};base64,${base64}`
      } catch (error) {
        console.error('Failed to download image from URL:', image.url, error)
        throw new ApiError(`Failed to download image from URL: ${image.url}`)
      }
    } else {
      throw new ApiError('No image data (b64_json or url) in response')
    }

    dataUrls.push(dataUrl)
    callback?.(dataUrl)
  }

  return dataUrls
}

/**
 * 辅助函数：构建 API URL，避免重复的路径段
 */
private buildImageGenerationUrl(): string {
  // 移除 apiHost 末尾的斜杠
  const baseUrl = this.options.apiHost.replace(/\/$/, '')
  
  // 确保路径以 /v1 开头
  let imagePath = '/v1/images/generations'
  
  // 如果 apiHost 已经包含 /v1，则使用空路径
  if (baseUrl.includes('/v1')) {
    imagePath = '/images/generations'
  }
  
  return `${baseUrl}${imagePath}`
}
```

**为什么需要重写 paint 方法？**

1. **AI SDK 的局限性**：Vercel AI SDK 的 `generateImage()` 内部使用 `detectMediaType` 函数，尝试用 `atob()` 解码响应。但如果 API 返回的是 URL，`atob()` 会失败
2. **自定义 API 的特殊性**：自定义 Provider 的 API Host 可能已经包含 `/v1` 路径，直接拼接会导致 `/v1/v1/` 重复
3. **兼容性**：不同模型返回格式不同（base64 或 url），需要手动处理

## 六、常见问题

### 6.1 atob 解码错误

**错误信息**:
```
Failed to execute 'atob' on 'Window': The string to be decoded is not correctly encoded.
```

**原因**:
- 第三方 API 返回的 `base64` 字段实际上是一个 URL
- 直接将 URL 当作 Base64 字符串进行解码

**解决方案**:
- 在使用 base64 字段前检查是否是 URL
- 如果是 URL，先下载图片再转换为 Base64

### 6.2 模型不显示

**原因**:
- 模型没有 'image' capability
- 使用了硬编码的模型列表

**解决方案**:
- 确保第三方模型正确设置了 capabilities
- 使用关键词检测作为回退方案

### 6.3 404 错误 - URL 重复 /v1/v1 ⚠️ 2025-04-27 新增

**错误信息**:
```
POST https://ai.comfly.chat/v1/v1/images/generations 404
```

**原因**:
- 用户的 API Host 是 `https://ai.comfly.chat/v1`（已包含 /v1）
- 代码直接拼接 `/v1/images/generations`，导致变成 `/v1/v1/images/generations`

**解决方案**:
- 添加 `buildImageGenerationUrl()` 方法
- 检查 apiHost 是否已包含 `/v1`
- 如果已包含，只添加 `/images/generations`

## 八、问题根源分析

### 8.1 AI SDK 的工作原理

Vercel AI SDK 的 `generateImage()` 函数：
1. 调用 provider 的 `imageModel().doGenerate()`
2. 内部使用 `detectMediaType` 函数检测返回数据的媒体类型
3. 尝试用 `atob()` 解码 base64 字段

**问题**：如果 API 返回的 `base64` 字段实际上是 URL（如 nano-banana），`atob()` 会失败。

### 8.2 为什么自定义 Provider 需要特殊处理

1. **继承关系**：`CustomOpenAI` 继承自 `AbstractAISDKModel`
2. **默认行为**：使用父类的 `paint()` 方法，该方法使用 AI SDK
3. **问题**：AI SDK 无法正确处理返回 URL 的模型
4. **解决方案**：在 `CustomOpenAI` 中重写 `paint()` 方法，直接调用 API

## 九、总结

### 9.1 架构优点

1. **分层清晰**: UI 层、业务逻辑层、抽象层、适配层分离
2. **可扩展性好**: 新增模型只需实现对应适配器
3. **容错性强**: 支持多种返回格式
4. **灵活性高**: 子类可以重写 paint 方法自定义处理逻辑

### 9.2 待优化点

1. 添加更完善的调试日志
2. 增加错误重试机制
3. 支持更多图片格式
4. 统一处理所有自定义 Provider 的 paint 方法

### 9.3 经验教训

1. **AI SDK 不是万能的**：对于某些特殊 API，直接调用可能更可靠
2. **URL 构建要小心**：处理用户输入的 API Host 时要注意路径拼接
3. **测试覆盖要全面**：不同模型的返回格式可能不同

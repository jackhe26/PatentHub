# 专利审查搭子功能说明

## 一、自动加载机制

### 1.1 加载流程图
```
应用启动
    ↓
useMyCopilots() Hook 挂载
    ↓
useEffect 触发（空依赖数组，只执行一次）
    ↓
setCopilots(async prev => {
    const current = await prev  // 获取当前存储的搭子
    return initializePatentCopilots(current)  // 初始化
})
    ↓
initializePatentCopilots():
    1. 提取专利搭子ID列表
    2. 提取当前存储的搭子ID列表  
    3. 检查是否已有专利搭子
    4. 如果没有 → 合并添加所有专利搭子
    5. 如果有 → 返回原列表
    ↓
搭子数据写入 localStorage
    ↓
UI 渲染搭子列表
```

### 1.2 核心代码逻辑
```typescript
// useCopilots.ts

// 1. 创建持久化 Atom
const myCopilotsAtom = atomWithStorage<CopilotDetail[]>(
  StorageKey.MyCopilots, 
  [],  // 初始值
  storage  // localStorage 适配器
)

// 2. 初始化函数
const initializePatentCopilots = async (currentCopilots) => {
  const patentIds = patentCopilots.map(p => p.id)
  const existingIds = currentCopilots.map(c => c.id)
  
  // 关键：检查是否已有专利搭子
  const hasPatentCopilots = patentIds.some(id => existingIds.includes(id))
  
  if (!hasPatentCopilots) {
    // 首次：追加所有专利搭子
    return [...currentCopilots, ...patentCopilots]
  }
  
  // 非首次：保持原样
  return currentCopilots
}

// 3. Hook 中触发初始化
export function useMyCopilots() {
  const [copilots, setCopilots] = useAtom(myCopilotsAtom)
  
  useEffect(() => {
    setCopilots(async (prev) => {
      const current = await prev
      return initializePatentCopilots(current)
    })
  }, [])  // ← 空数组：仅组件首次挂载时执行
  
  return { copilots, addOrUpdate, remove }
}
```

## 二、关键技术点

### 2.1 atomWithStorage (Jotai)
- 自动将状态同步到 localStorage
- 页面刷新后数据不丢失
- 支持异步更新

### 2.2 幂等性设计
```typescript
// 初始化只做一次判断：
// - 首次启动：currentCopilots = [] → 添加8个搭子
// - 再次启动：currentCopilots 已有8个 → 不重复添加
```
**为什么不会重复添加？**
- `some()` 检查 existingIds 中是否包含任一专利搭子ID
- 只要有一个专利搭子存在，就认为已初始化

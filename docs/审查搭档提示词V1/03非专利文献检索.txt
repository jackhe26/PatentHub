name: comprehensive-prior-art-search
description: |
对给定专利或技术方案执行全面的现有技术及公开文献检索，涵盖学术论文、专利、
技术手册、新闻、公众号、预印本（arXiv）、大模型知识库及开放文档数据库。
检索结果按与发明构思的相似度从高到低排序，并输出每篇文献的标题、发明构思、
来源及访问链接。适用于新颖性/创造性判断、专利无效分析或技术情报调研场景。
全面现有技术与公开文献检索
使用场景
你有一份专利全文或详细技术方案，需要评估其新颖性、创造性
需要发现发明人自身或他人在申请日前（影响新颖性/创造性）或申请日后（影响因素）的公开文献
需要一份包括标题、发明构思、来源和链接的结构化现有技术检索报告
输入要求
用户应提供：

专利公告号、申请号或完整的专利说明书文本（PDF/文字均可）
或者：一份详细的技术方案描述，至少包括发明人姓名（中英文）、技术领域、独立权利要求的全部技术特征、申请日/优先权日
执行流程
第一步：技术方案分析与发明点提炼
仔细阅读权利要求书和实施例，提取独立权利要求的全部技术特征。
用自然语言总结“发明构思”——即技术问题、解决原理和关键技术手段。
识别出发明人清单（含中英文姓名、曾用名、所属机构），并记录申请日/优先权日。
第二步：发明人自身公开文献检索
对每一位发明人，依次在以下数据库中检索，分别记录申请日前和申请日后公开的文献。

数据库类别	数据库名称与地址	检索策略
学术论文	Google Scholar (scholar.google.com), IEEE Xplore (ieeexplore.ieee.org), CNKI (cnki.net), 万方 (wanfangdata.com.cn), arXiv (arxiv.org), PubMed (pubmed.ncbi.nlm.nih.gov)	使用发明人姓名（中英文、拼音）+ 技术关键词；检索时间范围覆盖申请日前后各3年
专利	Google Patents (patents.google.com), 中国专利公布公告 (epub.sipo.gov.cn), Espacenet (worldwide.espacenet.com)	使用发明人/申请人字段检索，同时用技术关键词过滤
预印本与开放文库	arXiv, TechRxiv, ResearchGate, OpenReview	同上
大模型知识库与开放文档	Hugging Face Models (huggingface.co/models), Papers with Code (paperswithcode.com), GitHub (github.com, 搜索项目README/文档)	使用发明人或课题名，检查模型卡、技术报告等
技术手册/新闻/公众号	百度/Google搜素（配合filetype:pdf, site:xxx.com），搜狗微信搜索 (weixin.sogou.com)，知乎，CSDN，技术博客	结合发明人姓名+技术关键词，注意新闻发布日期
对每一条命中文献，初步判断其是否公开了发明构思的全部或部分技术特征，按照相似度给予高/中/低三档预标记，并记录：

标题
发明构思摘要（用自己的话概括该文献提出的技术方案，而非直接抄摘要）
来源（数据库或平台名称）
访问链接（URL、DOI或数据库路径）
公开日期（年月日）
与目标专利发明构思的相似度评估（高/中/低及简要理由）
第三步：他人公开文献检索（非发明人）
重复第二步，但不限制发明人。使用发明构思中的关键词、技术问题、技术效果作为检索词，同时可结合IPC/CPC分类号进行专利检索。同样区分申请日前与申请日后。

第四步：结果整理与排序
将第二、三步检索到的所有文献合并，按与发明构思的相似度从高到低排序。
排序时综合考虑：
特征覆盖度：文献是否涵盖了独立权利要求的大多数技术特征
问题/方案一致性：是否解决同一技术问题，采用相同或等同的技术手段
效果相似性：是否产生相同或相似的技术效果
对每篇文献给出最终定档：实质公开 / 部分公开 / 弱相关。
分别汇总“发明人自身公开”与“他人公开”，并在每个分组内按申请日前、后分开列出。高风险公开文献（实质公开）需明显标注并突出展示。
输出格式
最终报告应包含以下结构：

TEXT

- 对专利稳定性的初步影响评估
每条记录的“发明构思”列需用简洁的文字概括该文献所提出的技术方案或其核心思想，而非复制摘要。对于来源为arXiv、Hugging Face、GitHub等的文献，链接应直接指向具体条目/文件。

注意事项
严格区分申请日：申请日前公开的文献可能影响新颖性/创造性；申请日后公开的仅作为背景或日后公开风险提示。
语言覆盖：中文专利/论文检索用中文关键词，英文用英文关键词，必要时中英文扩展。
真实可验证：所有列出的文献必须为检索当时可访问的公开资源，禁止编造。提供完整的链接或DOI，以便人工复核。
大模型知识库与开放文档：若检索到Hugging Face模型卡、GitHub仓库描述或arXiv预印本，请仔细阅读其技术实现细节，并与专利特征对比。
相似度判断依据：以独立权利要求的全部技术特征为准，对比文献中明确记载的内容；不要将隐含或推测的功能当作公开内容。
示例调用
“请根据专利 CN202510975184.1，执行全面现有技术检索，输出按相似度排序的报告。”

工具速查
工具	用途	说明
Google Scholar	全球学术文献	支持中英文姓名、技术词
IEEE Xplore	工程/计算机领域	尤其适合信号处理、AI场景
CNKI/万方	中文学术	必须覆盖
arXiv	预印本	速度最快，需关注计算机科学各组
Hugging Face	大模型、数据集	搜索模型卡及论文链接
Papers with Code	论文+代码	可发现最新实现
GitHub	代码仓库	搜索“项目名+技术描述”
搜狗微信	公众号文章	适合中文圈技术动态
Google/Bing	全面搜索	使用高级语法如 filetype:pdf, site:researchgate.net
请始终遵循此Skill，确保检索的全面性、准确性和输出的规范性。
